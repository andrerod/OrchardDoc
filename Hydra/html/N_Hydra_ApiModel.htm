Reflected
The Reflected modeling attribute indicates that a model property should take its value from a property of its reflected instance. This is commonly used to set something like a Service.Name property to the corresponding Type.Name property of the reflected Type instance used to create the Service model. Note that the reflected value will not be used if the property already has a value.
SelfReference
The SelfReference modeling attribute indicates that we should search the current model's reflected instance for members attributed with the type of the property (or the element type if the property is a sequence/ collection). It can also be provided with a key property path which will be evaluated against the nested model if used with a dictionary. Here's an example model definition showing how to use MemberReference: public class Service : ModelObject { [Reflected] public string Name { get; set; } [SelfReference] public Documentation Documentation { get; private set; } } public class Documentation : ModelObject { public string Message { get; set; } public string Uri { get; set; } }
